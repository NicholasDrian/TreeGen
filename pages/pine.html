Details:

This tree generator is for mimicking the structure of pine trees. It generates a parametric trunk, parametric primary branches, and parametric secondary branches. primary branches are children of the trunk and secondary branches are children of primary branches. Because of the number of different branch types, this generator requires significantly more parameterization that the deciduous tree generator. The main functions of contained within the download are generators for all of the different branch types. Notably, there function for adding squiggles and foliage to branches.

Parameters & Outputs:

  • Generate Trunk
    • Root: This input takes in a point representing the location of the base of the tree.
    • Height: This input indicates the height of the trunk.
    • Squiggle Count: This input defines the number of control points of the trunk.
    • Squigglyness: This input defines the range of possible distortion of control points.
    • Seed: This input allows for repeatable randomness and for new trees to be generated.
    • Base Radius: This input defines the radius of the base of the trunk.
    • Profile Radii: These inputs define the radius of the trunk at evenly spaced intervals, relative to the base radius.
  • Primary Branches
    • Trunk Wire: This defines the trunk to add branches to.
    • Radial Branch Count: This input defines the number of branches per vertical level of the tree.
    • Width: This parameter defines the width of the widest part of the tree.
    • Base Branch Height: This input defines the height of the base of the lowest branch.
    • Angle: This input defines the angle of drupe of the branch.
    • Rotational Randomness: This input represents the range of possible rotations of each branch around the trunk. The units are degrees.
    • Vertical Randomness: This input should also be in the range [0-1] and represents the amount of randomness applied to the vertical placement of each branch
    • Length Randomness: This input should be from [0-1]. 0 represents uniform and exact length. 1 represents random length between 0 and 2x.
    • Seed: This input can be changed to create random yet repeatable results.
    • Profile Radii: The number of profile radii is equal to the number of vertical branch levels. Each radius corresponds to the average branch length at the corresponding vertical branch level.
  • Secondary Branches
    • Curves: These are the curves of the primary branches that spawn secondary branches.
    • Density: This input represents the number of secondary branches spawned per unit length of primary branch.
    • Drupe: This input defines the amount that the tip of a branch might drupe or angle upward.
    • Rotation: This parameter defines the range of random rotation.
    • Length: This parameter defines the length of secondary branches relative the length of their parent branch and relative to the position of the primary branch.
    • Seed: This input can be changed to generate a new and repeatable result.
  • Add Squiggles
    • Wires: This input should contain curves that represent branches.
    • Branch Squigglyness: This input defines the magnitude of distortion.
    • Squiggle Density: This input defines the number of control points of the output branch per unit length.
    • Seed: Allows for repeatable random results. Changing this input will create a new output.
