Details:

This tree generator consists of two main functions, "Deciduous Tree Generator" and "Foliage Generator". The first creates the trunk and the second generates a textured canopy.

Parameters & Outputs:

  • Deciduous Tree Generator Inputs
    • Branch Density: This input defines the number of branches that are generated per unit of surface area of the input canopy. CAUTION: Make sure to use very small values when the surface area of the canopy is large.
    • Canopy BRep: This input takes in one or more surfaces that represent the canopy of the tree. More specifically, these surfaces represent the places where branches will terminate.
    • Base Point: This input takes in a point that represents the base of the tree.
    • Factor: This input takes in a value between 0 and 1. This value defines the proportion of the distance that any branch segment will travel in the direction of its corresponding canopy. Each branch segment has its own factor randomly calculated to be between min_factor and max_factor. The best way to understand this input is through manipulation.
    • Factor: This input takes in a value between 0 and 1. This value defines the proportion of the distance that any branch segment will travel in the direction of its corresponding canopy. The best way to understand this input is through manipulation.
    • Noise: This input takes in a number representing the amount of randomness that is added to the position at which each branch terminates. If noise is 0, each branch will complete its path on the canopy surface. If noise is positive, branches will probably end up extending past the canopy, and negative values do the opposite.
    • Termination: Termination odds represent the proportion of the time a branch segment will have no children. This is for randomly thinning the tree tops. termination_depth is the minimum branch depth at which termination is allowed to happen.
    • Trunk Width: This input defines the trunk radius at the base of the tree. The radius along the tree diminishes proportional to the distance to the furthest child.
    • Help: This number takes in an integer and defines the corresponding instruction in the help output.
  • Deciduous Tree Generator Outputs
    • Help: Connect panel to this output to view text feedback.
    • Curves: This output contains the curvy skeleton of the tree that was generated.
    • Depths: This output contains the depths corresponding with each output curve. This output is for the purpose of foliage generation.
    • Tree: This output contains the actual tree geometry. Right click and select bake to save this output into Rhino.
  • Foliage Generator Inputs
    • Min Depth: This input defines the minimum branch depth that is allowed to spawn foliage. The depth of a given branch segment is the maximum of the depths of its children. Segments with no children have depth 0.
    • Curves: This input should be connected to the corresponding Tree Generator output.
    • Curves: This input should be connected to the corresponding Tree Generator output.
    • Density: This input defines the number of foliage textures generated per unit of branch length.
    • Randomness: This input defines how regularly spaced the foliage positioning is.
    • Length: The length of each textured polygon.
    • Width: The length of each textured polygon.
    • Random Rotation: This input defines the rotational randomness in radians.
    • Foliage Type: This input takes in integers 1 - 7 and produces different textures.
  • Foliage Generator Outputs
    • Outputs should remain plugged into the Custom Preview Materials function from the example document.
